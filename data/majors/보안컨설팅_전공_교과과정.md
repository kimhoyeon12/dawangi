## 전공간 중복 학점인정 교과목

### 📌 제도 설명
**각 개설학과 소속 학생이 해당학과 교과목을 수강하는 경우 중복 인정함**

**핵심 개념**
1. **이중 학점 인정**: 본전공 소속 학생이 해당 학과 교과목을 수강하면, 본전공과 보안컨설팅 전공 학점 모두 인정
2. **중복 수강 방지**: 같은 내용을 두 번 들을 필요 없음
3. **효율적 학점 관리**: 융합전공 이수 부담 경감

**활용 예시**
```
예시 1: 경영정보학과 학생 + 보안컨설팅 전공 이수 중
→ 경영정보학과 "프로그래밍 언어(6208069)" 수강 (필수 과목)
→ 경영정보학과 전공 학점 인정 ✅
→ 보안컨설팅 전공 필수 학점 인정 ✅
→ 한 과목으로 두 전공 동시 충족!

예시 2: 컴퓨터공학과 학생 + 보안컨설팅 전공 이수 중
→ 컴퓨터공학과 "데이터베이스시스템(5110025)" 수강
→ 컴퓨터공학과 전공 학점 인정 ✅
→ 보안컨설팅 전공 전공선택 학점 인정 ✅

예시 3: SW융합부전공 학생 + 보안컨설팅 전공 이수 중
→ SW융합부전공 "파이썬 프로그래밍(5114003)" 수강
→ SW융합부전공 학점 인정 ✅
→ 보안컨설팅 전공 전공선택 학점 인정 ✅
```

**💡 적용 원칙**: 교육과정표 및 "타학과(부, 전공) 전공선택 인정 교과목" 목록에 있는 모든 과목들은 각 개설학과 소속 학생이 수강할 경우 본전공과 보안컨설팅 전공 학점으로 중복 인정됩니다.

## 전공간 상호학점인정 대비표

### 📌 제도 설명
**서로 다른 전공의 유사 과목을 동일한 과목으로 인정하는 제도**

이 표는 특정 과목들이 다른 전공의 과목과 내용이 동일하여 상호 인정됨을 명시합니다.

### 상호인정 교과목

| 소프트웨어학과(전공) | 보안컨설팅 융합전공 | 비고 |
|---|---|:---:|
| **교과목번호**: 5111041<br>**교과목명**: 정보보호 | **교과목번호**: 4601033<br>**교과목명**: 정보보호개론 | 동일 |

**💡 의미**: 
- 소프트웨어학과의 "정보보호(5111041)"와 보안컨설팅 전공의 "정보보호개론(4601033)"은 동일한 과목입니다.
- 소프트웨어학과 학생이 "정보보호"를 들으면, 보안컨설팅 전공의 "정보보호개론"을 이수한 것으로 인정됩니다.
- 보안컨설팅 전공 학생이 "정보보호개론" 대신 소프트웨어학과의 "정보보호"를 들어도 인정됩니다.

**활용 예시**
```
예시 1: 시간표 충돌 해결
보안컨설팅 전공 "정보보호개론" 시간표가 안 맞음
→ 소프트웨어학과 "정보보호" 수강 ✅
→ 보안컨설팅 전공 학점으로 인정!

예시 2: 이중 학점 인정
소프트웨어학과 학생 + 보안컨설팅 전공 이수 중
→ 소프트웨어학과 "정보보호(5111041)" 수강
→ 소프트웨어학과 전공 학점 인정 ✅
→ 보안컨설팅 전공 "정보보호개론" 학점 인정 ✅
```

**※ 참고사항**
- 학점 표기 방식: `학점-이론시간-실습시간`
- 예: `3-2-2`는 3학점, 이론 2시간, 실습 2시간을 의미
- 인턴십 과목의 경우 `15-0-16주`는 15학점, 16주간 현장실습을 의미
- 보안컨설팅 전공은 정보보호 실무 전문가 양성을 위한 실습 중심 교육이 특징입니다
